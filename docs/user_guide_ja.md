# Em68030 ユーザーガイド

## クイックスタート

### インストール

1. アーカイブを任意のフォルダに展開します
2. `Em68030.exe` を実行します

> **注意**: 初回起動時に Windows Defender SmartScreen が未署名の実行ファイルをブロックする場合があります。
> 「詳細情報」→「実行」をクリックするか、exe を右クリック → プロパティ →「ブロックの解除」にチェックを入れてください。

### 初回起動 (NetBSD)

1. **Settings** (設定メニュー) を開きます
2. **Board Type** を "MVME147" に設定します
3. SCSI ディスクイメージを追加: **+ Add Disk** をクリックし、ディスクイメージを選択して SCSI ID を指定します
4. (任意) インストール用の CD-ROM ISO イメージを設定します
5. **OK** をクリックします
6. **File → Open ELF** (Ctrl+E) で NetBSD カーネル (`netbsd-GENERIC`) を読み込みます
7. **Run** (F5) を押して実行を開始します

OS インストールの詳細な手順については [Getting Started](getting_started_ja.md) を参照してください。

### 初回起動 (Linux)

1. **Settings** を開き、**Board Type** を "MVME147" に設定します
2. **Target OS** を "Linux" に設定します
3. **Command Line** を設定します (例: `root=/dev/sda1 console=tty0 console=ttyS0 earlyprintk`)
4. SCSI ディスクイメージを追加して **OK** をクリックします
5. **File → Open ELF** (Ctrl+E) で Linux カーネル (`vmlinux`) を読み込みます
6. **Run** (F5) を押します

### 逆アセンブリビュー (左上)

現在の PC 付近の逆アセンブルされた命令を表示します。

- **アドレスフィールド**: 16 進アドレスを入力して Enter を押すか「Go」をクリックして移動します
- **Follow PC**: 有効にすると、ビューが自動的にプログラムカウンタに追従します
- **右クリック コンテキストメニュー**: コピー、カーソルまで実行、アドレスに PC を設定

### レジスタパネル (右側)

CPU レジスタの表示と編集ができます。

- **D0–D7**: データレジスタ
- **A0–A7**: アドレスレジスタ (A7 はスタックポインタ)
- **PC**: プログラムカウンタ
- **SR**: ステータスレジスタ (X, N, Z, V, C, S, T の各フラグチェックボックス付き)
- **SSP**: スーパーバイザスタックポインタ
- **VBR**: ベクタベースレジスタ
- **FP0–FP7**: 浮動小数点レジスタ
- **FPCR/FPSR/FPIAR**: FPU 制御レジスタ

レジスタを編集するには: **Edit** をクリックし、値を変更してから **Apply** をクリックします (**Cancel** で破棄)。

### メモリダンプビュー (左下)

メモリの内容を 16 進数と ASCII で表示します。

- **アドレスフィールド**: 16 進アドレスを入力して移動します
- **サイズフィールド**: 表示するバイト数
- **編集**: **Edit** をクリックし、16 進値を変更してから **Apply** をクリックします

**ナビゲーションキー** (編集モード時):
| キー | 動作 |
|------|------|
| 矢印キー | セル間を移動 |
| Tab / Shift+Tab | 次 / 前のセルへ移動 |
| Enter | 編集を確定 |
| Escape | 編集をキャンセル |

### ステータスバー

メインウィンドウの下部に実行時情報を表示します:

- **実行状態**: Running: True/False
- **ネットワークモード**: Net: Virtual / NAT / TAP (Bridge)
- **JIT 状態**: JIT: ON / OFF
- **トレース状態**: [TRACE ON] (有効時)

## シリアルコンソールウィンドウ

シリアル I/O 用の VT100 互換ターミナルエミュレータです。

### 機能
- スクロールバックバッファ (設定可能、デフォルト 2000 行)
- ウィンドウのリサイズ (ターミナルの桁数と行数を自動調整)
- テキスト選択とコピー
- クリップボード貼り付け
- 正規表現対応のテキスト検索
- ソフトラップ対応コピー (表示上の折り返しはクリップボードに改行を挿入しない)

### キーボードショートカット

| ショートカット | 動作 |
|----------------|------|
| Ctrl+C | 選択テキストをコピー (選択がない場合は ETX を送信) |
| Ctrl+V | クリップボードのテキストをゲストに貼り付け |
| Ctrl+Shift+F | 検索バーを開く |
| F3 | 次の一致を検索 (検索モード) |
| Shift+F3 | 前の一致を検索 (検索モード) |
| Escape | 検索バーを閉じる (検索モード) |
| Ctrl+A–Z | ゲストにコントロールコードを送信 |

### 検索

**Ctrl+Shift+F** を押すと、ウィンドウ下部に検索バーが表示されます。

- 検索語を入力して **Enter** または **F3** を押すと次の一致を検索します
- **Shift+F3** で前の一致を検索します
- **`.*`** トグルボタンで正規表現モード (大文字小文字を区別しない) を有効にします
- マッチカウンターに現在の一致番号と合計が表示されます (例: "3/15")
- **Escape** を押すと検索バーを閉じます

> **注意 (C# WPF)**: 正規表現モードでは `(?-i)` などのインライン修飾子による
> 大文字小文字を区別する検索がサポートされています。
>
> **注意 (C++ WinUI3)**: ECMAScript 正規表現構文が使用されます。インライン修飾子はサポートされていません。

### コンテキストメニュー

コンソール内で右クリックすると以下にアクセスできます:
- **Copy** (Ctrl+C)
- **Paste** (Ctrl+V)
- **Select All** (Ctrl+A)

## フレームバッファウィンドウ

エミュレートされたフレームバッファ出力を表示します。設定でフレームバッファを有効にし、
ゲスト OS で `em68030fb` カーネルモジュールをロードする必要があります。

### 機能
- リアルタイム VRAM 表示 (8/16/32 ビットカラー)
- キーボード入力キャプチャ (フォーカス時)
- マウス入力 (X Window System 互換の絶対座標)

### キーボードショートカット

| ショートカット | 動作 |
|----------------|------|
| Ctrl+Shift+V | クリップボードのテキストをキーイベントとして貼り付け |

### マウス入力

フレームバッファウィンドウ内のマウス位置はゲスト画面座標に直接マッピングされます
(絶対位置指定)。ポインタのグラブは不要です。左、右、中ボタンのクリックは
ゲストに転送されます。

## ブレークポイントウィンドウ

デバッグ用の CPU ブレークポイントを管理します。

- **ブレークポイントの設定**: 逆アセンブリビューでアドレスをダブルクリックします
- **全クリア**: **Clear All** ボタンをクリックします
- **ブレークポイントまで実行**: F5 を押すと、ブレークポイントアドレスに到達した時点で実行が停止します

## 設定リファレンス

設定は `%LOCALAPPDATA%\Em68030_CsWPF\appsettings.json` に保存されます。
設定ダイアログは **General**、**MVME147**、**Advanced** の3つのタブで構成されています。

### General タブ

#### ボードタイプ

| 設定 | 値 | デフォルト | 説明 |
|------|-----|-----------|------|
| Board Type | Generic, MVME147 | Generic | エミュレートするボードを選択 |

#### メモリ

| 設定 | 範囲 | デフォルト | 説明 |
|------|------|-----------|------|
| Memory Size (MB) | 4–4096 | 48 | ゲスト RAM サイズ |

> **注意**: メモリサイズの変更はカーネルイメージを再読み込みした後に有効になります。

#### I/O デバイス

| 設定 | デフォルト | 説明 |
|------|-----------|------|
| Console Device Enabled | On | シリアルコンソールを有効にする |
| Console Base Addr | 0x00FF0000 | コンソール MMIO アドレス |
| Scrollback Lines | 2000 | コンソールのスクロールバックバッファサイズ |
| Terminal Size | 80 x 24 | コンソールの桁数と行数 |
| HDD Device Enabled | On | HDD コントローラを有効にする |
| HDD Base Addr | 0x00FF1000 | HDD MMIO アドレス |

#### 表示

| 設定 | デフォルト | 説明 |
|------|-----------|------|
| Font Family | Consolas | コンソールと逆アセンブリのフォント |
| Font Size | 14.0 | フォントサイズ (ポイント) |

### MVME147 タブ

このタブは Board Type が MVME147 の場合のみ有効です。

#### MVME147 設定

| 設定 | 説明 |
|------|------|
| ROM Image | MVME147 ROM イメージのパス (任意) |

#### Target OS

| 設定 | 値 | デフォルト | 説明 |
|------|-----|-----------|------|
| Operating System | NetBSD, Linux | NetBSD | ゲスト OS の種類 |

**NetBSD 設定:**

| 設定 | 説明 |
|------|------|
| Kernel Image | NetBSD カーネルのパス（指定時は起動時に自動読み込み） |
| Boot Partition | a または b |

**Linux 設定:**

| 設定 | 説明 |
|------|------|
| Kernel Image | Linux カーネルのパス（指定時は起動時に自動読み込み） |
| Command Line | カーネルコマンドライン |

> **注意**: NetBSD と Linux のカーネルイメージは個別に保存されるため、OS を切り替えても
> カーネルパスを再設定する必要はありません。

#### SCSI ディスク

| 設定 | 説明 |
|------|------|
| Disk Image Path | RAW SCSI ディスクイメージファイルのパス |
| SCSI ID | 0–6 (各ディスクは一意の ID が必要) |
| + Add Disk | SCSI ディスクを追加 |
| Create | 新しい空のディスクイメージを作成 (サイズは MB 単位) |

> **注意**: SCSI ディスクのパス変更はゲスト OS を再起動した後に有効になります。
> CD-ROM ISO イメージはいつでも交換できます。

#### ネットワーク

| 設定 | 値 | デフォルト | 説明 |
|------|-----|-----------|------|
| Network Mode | Virtual, NAT, TAP (Bridge) | Virtual | ネットワークバックエンド |
| TAP Adapter | (ドロップダウン) | | TAP-Windows アダプタ (TAP モードのみ) |
| Gateway IP | IP アドレス | 10.0.2.2 | NAT ゲートウェイアドレス (NAT モードのみ) |
| Gateway MAC | MAC アドレス | 52:54:00:12:34:56 | NAT ゲートウェイ MAC (NAT モードのみ) |

> **注意**: ネットワークモードの変更はカーネルイメージを再読み込みした後に有効になります。
> ゲスト OS の設定については以下を参照してください:
> - [NAT ネットワーク セットアップガイド](setup_nat_network_ja.md) — NAT モードのゲスト設定 (Linux/NetBSD)
> - [TAP ブリッジ セットアップガイド](setup_tap_bridge_ja.md) — TAP ブリッジモード（TAP-Windows ドライバが必要）

#### フレームバッファ

| 設定 | 値 | デフォルト | 説明 |
|------|-----|-----------|------|
| Enable | On/Off | Off | 仮想フレームバッファを有効にする |
| Resolution | 320x240 ～ 1920x1080 | 640x480 | 表示解像度 |
| Bits per Pixel | 8, 16, 32 | 16 | 色深度 |

> **注意**: VRAM は RAM の上端に自動的に配置されます。

### Advanced タブ

#### パフォーマンス

| 設定 | デフォルト | 説明 |
|------|-----------|------|
| Enable JIT Compiler | Off | レジスタのみのブロック用の実験的 JIT |
| Min Block Length | 3 | JIT コンパイルの最小命令数 |
| Compile Threshold | 32 | ブロックがコンパイルされるまでの実行回数 |

#### デバッグ

| 設定 | デフォルト | 説明 |
|------|-----------|------|
| Trace ボタンを表示 | OFF | ツールバーに Trace ボタンを表示 |

Trace ボタンが表示されている状態でクリックすると、命令トレースの出力が切り替わります。
トレースは `%LOCALAPPDATA%\Em68030_CsWPF\tracelog.txt` に出力されます。
トレースには命令実行、例外ベクター、syscall エントリが含まれます。

> **注意:** トレースはエミュレーション速度を大幅に低下させ、大きなログファイルを生成します。
> 特定の問題のデバッグ時のみ使用してください。

## コマンドライン引数

| 引数 | 例 | 説明 |
|------|-----|------|
| `--lang=xx-XX` | `--lang=en-US` | UI 言語を上書きする (例: `en-US`, `ja-JP`) |

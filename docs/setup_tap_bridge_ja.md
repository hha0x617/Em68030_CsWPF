# TAP ブリッジネットワーク セットアップガイド

このガイドでは、Em68030 エミュレータで TAP ブリッジネットワークを設定する方法を説明します。
TAP ブリッジモードはゲストをホスト LAN に直接接続し、ゲストがネットワーク上で
独自の IP アドレスを持つことを可能にします。

## 概要

| モード | 説明 | 用途 |
|--------|------|------|
| Virtual (Echo Server) | 内蔵エコーサーバー、ホストネットワークなし | ネットワークスタックのテスト |
| NAT (Host Network) | エミュレータ経由のユーザーモード NAT | ホスト設定不要でインターネットアクセス |
| **TAP (Bridge)** | **ホスト LAN への直接 L2 ブリッジ** | **完全なネットワーク参加、DHCP、サーバーホスティング** |

## 前提条件

- Windows 10/11
- OpenVPN TAP-Windows ドライバがインストール済み
- DHCP サーバーのある LAN に接続された物理ネットワークアダプタ（DHCP モードの場合）

## ステップ 1: TAP-Windows ドライバのインストール

TAP-Windows ドライバは OpenVPN インストーラに含まれています。

1. [OpenVPN Community Downloads](https://openvpn.net/community-downloads/) から OpenVPN インストーラをダウンロード
2. インストーラを実行
3. インストール中に **TAP-Windows** コンポーネントが選択されていることを確認
4. インストールを完了し、求められた場合は再起動

**確認:** ネットワーク接続（`ncpa.cpl`）を開きます。「TAP-Windows Adapter V9」
（または類似の名前）という新しいアダプタが表示されるはずです。

## ステップ 2: Windows ネットワークブリッジの設定

ゲストがホスト LAN と通信できるように、TAP アダプタを物理ネットワークアダプタと
ブリッジする必要があります。

1. ネットワーク接続を開く: `Win+R` を押して `ncpa.cpl` と入力し、Enter を押す
2. 両方のアダプタを選択:
   - **TAP-Windows Adapter V9** をクリック
   - `Ctrl` を押しながら **物理ネットワークアダプタ**（例: 「イーサネット」または「Wi-Fi」）をクリック
3. 選択したいずれかのアダプタを右クリック
4. **「ブリッジ接続」** をクリック
5. Windows がブリッジを作成するまで待つ（新しい「ネットワーク ブリッジ」アダプタが表示される）

> **注意:** ブリッジの作成中、ホストネットワークが一時的に切断される場合があります。
> ブリッジの設定が完了すると、接続は自動的に復元されます。

> **注意:** 一部のシステムでは Wi-Fi アダプタがブリッジをサポートしない場合があります。
> Wi-Fi でブリッジに失敗する場合は、有線イーサネット接続を使用してください。

## ステップ 3: エミュレータの設定

1. エミュレータの設定を開く（設定メニューまたはツールバーボタン）
2. **Network Mode** ドロップダウンで **「TAP (Bridge)」** を選択
   - このオプションがグレーアウトしている場合、TAP-Windows ドライバがインストールされていません（ステップ 1 を参照）
3. **TAP Adapter** ドロップダウンで TAP アダプタを選択
4. OK をクリックして保存

## ステップ 4: ゲスト OS の設定

### Linux

#### オプション A: DHCP（推奨）

ゲスト上の `/etc/systemd/network/10-eth0.network` を編集:

```ini
[Match]
Name=eth0

[Network]
DHCP=yes
```

設定を適用:

```bash
systemctl restart systemd-networkd
```

または `dhcpcd` を直接使用:

```bash
dhcpcd eth0
```

#### オプション B: 静的 IP アドレス

ゲスト上の `/etc/systemd/network/10-eth0.network` を編集:

```ini
[Match]
Name=eth0

[Network]
Address=192.168.1.100/24
Gateway=192.168.1.1
DNS=192.168.1.1
```

アドレスはお使いのネットワークに適した値に置き換えてください。

設定を適用:

```bash
systemctl restart systemd-networkd
```

### NetBSD

#### オプション A: DHCP（推奨）

ゲスト上の `/etc/rc.conf` を編集:

```
ifconfig_le0=dhcp
```

設定を適用:

```
# /etc/rc.d/dhcpcd restart
```

#### オプション B: 静的 IP アドレス

ゲスト上の `/etc/rc.conf` を編集:

```
ifconfig_le0="inet 192.168.1.100 netmask 255.255.255.0"
defaultroute="192.168.1.1"
```

`/etc/resolv.conf` を編集:

```
nameserver 192.168.1.1
```

アドレスはお使いのネットワークに適した値に置き換えてください。

設定を適用:

```
# /etc/rc.d/network restart
```

## ステップ 5: 接続の確認

1. **IP アドレスの確認:**
   ```bash
   # Linux
   ip addr show eth0

   # NetBSD
   ifconfig le0
   ```

2. **接続テスト:**
   ```bash
   ping -c 3 8.8.8.8
   ```

3. **DNS 解決のテスト:**
   ```bash
   ping -c 3 google.com
   ```

## トラブルシューティング

### TAP アダプタが「ネットワーク ケーブルが接続されていません」と表示される

エミュレータが TAP デバイスに接続されていません。以下を確認してください:
- エミュレータの設定で「TAP (Bridge)」が正しいアダプタとともに選択されている
- カーネルイメージがロードされ、ゲスト OS が実行中である
- エミュレータが起動時にデバイスを開くと TAP アダプタのリンクがアクティブになる

### 設定で「TAP (Bridge)」を選択できない

TAP-Windows ドライバがインストールされていないか、TAP アダプタが検出されませんでした。
TAP-Windows コンポーネントを含めて OpenVPN を再インストールしてください。

### DHCP がアドレスを割り当てない

- Windows ブリッジが設定されていることを確認（ステップ 2）
- ホスト上のブリッジアダプタにネットワーク接続があることを確認
- LAN 上に DHCP サーバーが利用可能であることを確認（通常はルーター）
- デバッグ出力には `dhcpcd -d eth0` を試す

### ゲストがゲートウェイに ping できるがインターネットに接続できない

- DNS 設定を確認: `cat /etc/resolv.conf`
- IP アドレスに直接 ping を試す: `ping 8.8.8.8`
- IP は動作するが DNS が動作しない場合、ネームサーバーを追加:
  ```bash
  echo "nameserver 8.8.8.8" > /etc/resolv.conf
  ```

### ブリッジがホストネットワークを切断する

ブリッジ作成時に短時間切断されるのは正常です。ホストの接続が永続的に失われる場合は、
ブリッジを削除（右クリック → 「ブリッジから削除」）して再作成してください。

### Wi-Fi アダプタをブリッジできない

一部の Wi-Fi ドライバはブリッジモードをサポートしていません。有線イーサネット接続を使用するか、
代わりに NAT モードを使用してください。

## NAT モードとの比較

| 機能 | NAT モード | TAP ブリッジモード |
|------|-----------|-------------------|
| ホスト設定 | 不要 | TAP ドライバ + ブリッジ設定 |
| ゲスト IP アドレス | プライベート (10.0.2.x) | LAN アドレス（DHCP または静的） |
| LAN からアクセス可能 | いいえ | はい |
| LAN からの DHCP | いいえ | はい |
| プロトコルサポート | TCP, UDP, ICMP | すべて（L2 ブリッジ） |
| 管理者権限 | 不要 | 不要（セットアップ後） |
| パフォーマンス | やや高いオーバーヘッド | 低いオーバーヘッド（直接ブリッジ） |
